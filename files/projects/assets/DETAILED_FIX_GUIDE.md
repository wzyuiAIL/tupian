# 🔧 Failed to fetch 错误详细解决方案

## ❌ 问题描述

即使API地址和Token都已配置，仍然出现：
```
错误: Failed to fetch
```

---

## 🔍 根本原因分析

### "Failed to fetch" 是什么意思？

这是一个通用的网络错误，表示浏览器无法成功发送请求到服务器。可能的原因包括：

1. **CORS跨域限制**（最常见）⚠️
2. API地址格式错误
3. Token格式错误
4. 网络连接问题
5. API服务器问题

---

## ✅ 解决方案（按顺序尝试）

我已为您创建了一个专门的调试工具！

#### 步骤：

4. **运行测试3：完整测试**

- ✅ 具体的错误信息

### 方案2：检查API地址格式

#### 正确的API地址格式：

```
https://px5r5j8mt8.coze.site/run
```

**要点**：
- ✅ 必须包含 `https://`
- ✅ 必须包含 `.coze.site` 或 `.coze.cn`
- ✅ 必须以 `/run` 结尾
- ✅ 不要有多余的空格或换行

#### 错误的API地址格式：

```
❌ https://px5r5j8mt8.coze.site         (缺少 /run)
❌ px5r5j8mt8.coze.site/run           (缺少 https://)
❌ https://px5r5j8mt8.coze.site/run/   (多余的 /)
❌  https://px5r5j8mt8.coze.site/run   (开头有空格)
```

---

### 方案3：检查Token格式

#### 正确的Token格式：

```
pat_v2_xxxxxxxxxxxxxx_xxxxxxxxxxxxxx_xxxxxxxxxxxxxx_xxxxxxxxxxxxxx
```

**要点**：
- ✅ 必须以 `pat_v2_` 开头
- ✅ 通常包含4段，用下划线分隔
- ✅ 不要有多余的空格或换行
- ✅ 完整复制，不要遗漏字符

#### 错误的Token格式：

```
❌ pat_xxxxxxxxxxxxxx                  (缺少 v2_)
❌ pat_v2_xxxxxxxxxxxxxx              (不完整)
❌  pat_v2_xxxxxxxxxxxxxx             (开头有空格)
❌ pat_v2_xxxxxxxxxxxxxx              (结尾有空格)
```

---

### 方案4：解决CORS跨域问题

#### 什么是CORS？

CORS（跨源资源共享）是浏览器的安全机制，阻止网页访问其他域名的API。

#### 如何判断是CORS问题？

1. 打开浏览器控制台（F12）
2. 切换到"Console"标签
3. 点击"生成图片"
4. 如果看到类似以下的错误：

```
Access to fetch at 'https://xxx.coze.site/run' from origin 'null' has been blocked by CORS policy
```

这就是CORS问题！

#### 解决CORS问题的方法：

##### 方法1：使用本地服务器（推荐）⭐

```bash
# 使用Python启动本地HTTP服务器
cd assets
python -m http.server 8080

# 然后在浏览器中访问
http://localhost:8080/web_sdk_index.html
```

##### 方法2：使用浏览器扩展

安装CORS扩展（Chrome/Edge）：
1. 打开Chrome Web Store
2. 搜索"CORS Unblock"或"Allow CORS"
3. 安装扩展
4. 启用扩展
5. 刷新页面，重新尝试

##### 方法3：使用VS Code的Live Server

1. 安装Live Server扩展
2. 右键HTML文件
3. 选择"Open with Live Server"
4. 在浏览器中访问

##### 方法4：使用无CORS限制的浏览器

使用开发者工具：
1. 打开Chrome
2. 右键快捷方式 → 属性
3. 在"目标"中添加：`--disable-web-security --user-data-dir="C:/chrome_dev"`
4. 重启Chrome
5. 打开HTML文件

⚠️ **警告**：此方法会降低浏览器的安全性，仅用于测试！

### 方案5：检查网络连接

#### 测试网络连接：

1. **测试能否访问Coze平台**
   - 在浏览器中访问：https://www.coze.cn
   - 如果无法访问，检查网络连接

2. **测试API地址是否可达**
   - 在浏览器中直接访问API地址
   - 应该看到JSON响应或错误信息

3. **测试防火墙/代理**
   - 关闭VPN
   - 关闭防火墙
   - 使用无代理模式

---

### 方案6：检查API状态

#### 确认API是否可用：

1. **检查工作流是否已发布**
   - 打开Coze工作流
   - 确认工作流状态为"已发布"

2. **检查接入方式是否启用**
   - 点击"查看接入方式"
   - 确认接入方式已启用

3. **检查Token是否过期**
   - 如果Token已过期，需要重新生成
   - 在Coze平台删除旧的接入方式
   - 创建新的接入方式
   - 使用新的Token
