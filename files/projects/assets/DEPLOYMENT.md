# Web界面部署说明

## 📋 前置条件

1. Python 3.12 或更高版本
2. 已安装项目依赖（`uv sync`）
3. 已完成工作流配置

## 🚀 快速部署

### 1. 启动Web服务器

```bash
cd scripts
./start_web.sh
```

或使用命令：

```bash
cd src
python web_app.py
```

### 2. 访问Web界面

启动成功后，浏览器访问以下地址：

- **本地访问**: http://localhost:5001
- **局域网访问**: http://[你的IP地址]:5001

## 📱 使用流程

### 第一步：上传图片

1. 点击"点击上传图片1"按钮，选择第一张产品图片
2. 点击"点击上传图片2"按钮，选择第二张产品图片
3. 预览图会显示在页面上
4. 如需更换，点击图片右上角的"✕"按钮删除

### 第二步：选择风格

- **智能自适应** 🤖：AI自动分析图片，智能选择最佳风格（推荐）
- **暖色系** 🔥：温馨舒适，Z世代最爱
- **冷色系** ❄️：现代清新，高级冷淡风
- **中性色** ⚪：高端百搭，极简专业

### 第三步：设置导出路径（可选）

如需将图片保存到本地：
- 输入路径，如：`/tmp/product_images_export`
- 留空则只返回URL，不保存到本地

### 第四步：生成图片

1. 点击"生成图片"按钮
2. 等待2-5分钟（图片生成需要时间）
3. 页面会显示实时进度
4. 生成完成后自动显示结果

### 第五步：下载图片

- **单张下载**：点击每张图片下方的"下载"按钮
- **批量下载**：点击右上角的"导出所有图片"按钮

### 第六步：重新生成（可选）

- 点击"重新生成"按钮即可生成全新的图片
- 或选择不同的风格再次生成

## 🔧 高级配置

### 修改端口号

默认端口：5001

修改方法：

```bash
WEB_PORT=8080 python src/web_app.py
```

或修改启动脚本：

```bash
export WEB_PORT=8080
cd src
python web_app.py
```

### 生产环境部署

使用Gunicorn：

```bash
pip install gunicorn
cd src
gunicorn -w 4 -b 0.0.0.0:5001 web_app:app
```

使用Docker：

```bash
docker build -t product-image-generator .
docker run -p 5001:5001 product-image-generator
```

### Nginx反向代理

配置示例：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:5001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 🐛 常见问题

### Q1: 无法启动Web服务器？

A: 检查端口是否被占用：

```bash
lsof -i :5001
```

修改端口号：

```bash
WEB_PORT=8080 python src/web_app.py
```

### Q2: 上传图片后没有预览？

A: 检查：
- 文件格式是否为PNG/JPG/JPEG/GIF
- 文件大小是否超过16MB
- 浏览器是否支持文件上传

### Q3: 生成图片时提示错误？

A: 查看：
- 上传的图片是否符合要求
- 工作流是否正常配置
- 控制台日志输出

### Q4: 下载的图片打不开？

A: 检查：
- 网络连接是否正常
- 图片URL是否有效
- 浏览器是否支持图片格式

### Q5: 导出路径无效？

A: 确保：
- 输入的路径存在
- 有写入权限
- 建议使用 `/tmp` 目录
