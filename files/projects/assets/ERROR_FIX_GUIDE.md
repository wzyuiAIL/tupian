# 🔧 Failed to fetch 错误解决指南

## ❌ 问题描述

您打开了HTML文件，上传了图片，但是点击"生成图片"后出现以下错误：

```
错误: Failed to fetch
```

---

## 🔍 问题原因

**原因**：没有配置Coze工作流的API地址和Token。

从您的截图可以看出，页面顶部没有显示"⚙️ API配置"区域，说明：

1. 您没有输入API地址
2. 您没有输入API Token
3. 系统无法连接到Coze工作流服务

---

## ✅ 解决方法

### 第1步：打开Coze平台

在浏览器中打开Coze平台，找到"无水印产品图工作流"。

### 第2步：获取API信息

在工作流页面右上角，找到并点击 **"查看接入方式"** 按钮。

在弹出的窗口中，您会看到：

```
API地址：https://xxxxx.coze.site/run
Token：pat_xxxxxxxxxxxxxx
```

### 第3步：配置API

**如果使用新版本HTML（已修复）：**

1. 滚动页面到顶部
2. 找到红色的警告区域："⚠️ 重要提示：请先配置API"
3. 继续向下滚动到"⚙️ API配置（必填）"区域
4. 在"工作流API地址"输入框中，粘贴API地址
5. 在"请输入API Token"输入框中，粘贴Token
6. 当两个输入框都有内容后，红色警告会自动消失

**如果使用旧版本HTML：**

1. 滚动页面到顶部
2. 找到"⚙️ API配置"区域（应该在标题下方）
3. 在第一个输入框中，粘贴API地址
4. 在第二个输入框中，粘贴Token

### 第4步：保存配置（自动保存）

新版本HTML会自动保存配置，无需额外操作。

旧版本HTML可能需要刷新页面或重新上传图片。

### 第5步：重新生成

1. 确认两个输入框都有内容
2. 点击"生成图片"按钮
3. 等待2-5分钟
4. 查看生成的图片

---

## 📸 获取API详细步骤

### 1. 打开Coze工作流页面

在Coze平台找到您的"无水印产品图工作流"

### 2. 点击"查看接入方式"

位置：页面右上角，通常是一个"查看接入方式"或"接入"按钮

### 3. 复制API信息

在弹出的窗口中，您会看到类似以下内容：

```
API地址
https://px5r5j8mt8.coze.site/run

Token
pat_v2_xxxxxxxxxxxxxx_xxxxxxxxxxxxxx_xxxxxxxxxxxxxx_xxxxxxxxxxxxxx
```

**重要**：
- 复制完整的API地址（包含https://）
- 复制完整的Token（包含pat_v2_前缀）
- 不要复制多余的空格或换行

❌ 包含多余空格

❌ 复制不完整
API地址：https://px5r5j8mt8.coze.site

## 🔍 如何确认配置成功

### 方法1：检查警告提示

新版本HTML：
- 配置前：显示红色警告"⚠️ 重要提示：请先配置API"
- 配置后：红色警告自动消失

### 方法2：检查输入框内容

- API地址输入框：应该显示完整的URL
- Token输入框：应该显示圆点（表示已输入密码）

### 方法3：尝试生成图片

点击"生成图片"按钮：
- ✅ 成功：显示加载动画，开始生成
- ❌ 失败：仍然显示"Failed to fetch"

## ⚠️ 常见问题

### Q: 找不到"查看接入方式"按钮？

A:
- 确保您在工作流页面
- 检查右上角是否有"接入"、"查看接入方式"等按钮
- 可能需要是工作流的创建者或有权限才能查看

### Q: Token显示"暂无Token"？

A:
- 您需要先在工作流中配置接入方式
- 可能需要创建新的接入方式
- 确保您有工作流的编辑权限

### Q: 配置后仍然报错？

A:
1. 检查API地址是否完整（包含https://）
2. 检查Token是否完整（包含pat_v2_前缀）
3. 检查网络连接
4. 尝试刷新页面后重新配置
5. 联系工作流创建者确认API是否可用

### Q: 警告提示不消失？

A:
1. 确认两个输入框都有内容
2. 刷新页面
3. 检查浏览器控制台是否有错误
